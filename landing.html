HCI User Study

Form Validation Comparison Study

Welcome!

Thank you for participating in this user study. We are comparing three different approaches to form validation and error handling.

Each form collects the same information but uses a different validation method. Your completion time, error encounters, and submission attempts will be recorded for analysis.

Instructions

  • Complete each form below in any order you prefer
  • Fill in all required fields with valid information
  • The timer starts when you open each form
  • After successful submission, you'll see your completion metrics
  • No personal data is stored - this is for research purposes only

Version A: Inline Validation

Errors are shown immediately below each field as you type or move to the next field. Real-time feedback helps you correct mistakes as you go.

Start Form A

Version B: Error Summary

All validation errors are displayed in a summary at the top of the form after you click Submit. Review all errors at once before making corrections.

Start Form B

Version C: Auto-correct/Suggestions

The form suggests corrections for common mistakes and allows you to apply them with one click. Helpful hints guide you toward valid input.

Start Form C
